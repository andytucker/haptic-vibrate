🧭 How the settings logic works

Short version: target, feel, preview, save.

  1. Add a rule for a selector like .buy-now, .menu-link, or a component-specific class.
  2. Choose a preset for a known pattern or switch to Custom… to define a timing sequence yourself.
  3. Use the generated pattern class when you want reusable markup without repeating selector-specific configuration.
  4. Test inline or through the sidebar tester to confirm the exact feel before saving.
  5. Enable debug mode to QA on desktop with the same fallback behavior used in the plugin admin.

🏷️ Generated classes, used like a pro

The landing now explains the reusable class workflow instead of hiding it behind marketing copy.

Each preset automatically maps to a predictable class name such as .haptic-vibrate-success or .haptic-vibrate-double_tap. That’s ideal for blocks, page builders, custom templates, and snippets shared across multiple screens.

.haptic-vibrate-success Confirmation and completed actions
.haptic-vibrate-heartbeat High-emphasis CTAs and hero buttons
.haptic-vibrate-custom-3 Special patterns for one-off interfaces
